## 搜索模式 UI 修复计划

您反馈的“搜索模式下样式和图标异常”问题，是因为在最近的树状结构重构中，我主要优化了 `renderTreeView`，而没有完全同步更新搜索结果的渲染函数 `doRender`。

为了解决这个问题，我将进行以下修复：

### 1. 统一样式 (content.js)
- **容器结构**: 将搜索结果 (`doRender`) 的 HTML 结构调整为与树状节点 (`renderNode`) 一致。
- **图标对齐**: 
    - 为搜索结果项添加与树状结构一致的左侧缩进占位符（`div.indent`），确保它们在视觉上与根目录对齐。
    - 统一使用新的 `img.favicon` 样式类。

### 2. 布局优化
- **间距调整**: 减小搜索结果的上下内边距，使其与侧边栏的紧凑风格保持一致。
- **URL 显示**: 优化搜索结果中 URL 的显示方式，避免在窄侧边栏中过度换行。

### 3. 图标逻辑修复
- 确保搜索结果中的 Favicon 加载逻辑与树状视图完全相同，包括 Chrome 特权 API 调用和错误回退机制。
