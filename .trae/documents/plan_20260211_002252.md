# 优化设置面板层级与透明度计划

## 问题分析
用户反馈打开设置模式时，由于2层面板重叠（主面板+设置面板），且两层都应用了半透明背景和毛玻璃效果，导致叠加后变得不透明，失去了预期的清透感。

## 解决方案
**核心思路**：当打开设置（或编辑/删除）面板时，隐藏主面板的内容，让设置面板直接复用主面板容器的背景和毛玻璃效果，从而避免双重叠加。

### 1. CSS 样式调整 (`content.js`)
- **Modal 状态类**: 定义 `.mode-settings` (及 `.mode-edit`, `.mode-delete`) 类，当该类加在父容器 `#fast-bookmark-modal` 上时：
    - 隐藏主界面的子元素（Header, Search, List, EmptyState, Footer）。
    - 设置面板 (`#fast-bookmark-settings-modal` 等) 的背景设为透明，移除 `backdrop-filter`，只保留内容布局。
- **通用样式**: 确保所有覆盖层 Modal (`settings`, `edit`, `delete`) 在显示时都遵循此逻辑。

### 2. JavaScript 逻辑调整 (`content.js`)
- **打开设置时**:
    - 给 `#fast-bookmark-modal` 添加 `.mode-settings` 类。
    - 显示 `#fast-bookmark-settings-modal` (现有的 `display: flex` 逻辑保持不变)。
- **关闭设置时**:
    - 移除 `#fast-bookmark-modal` 的 `.mode-settings` 类。
    - 隐藏 `#fast-bookmark-settings-modal`。
- **编辑/删除同理**: 为 `openEditModal` / `openDeleteModal` 及其关闭操作添加对应的类名切换逻辑。

## 实施步骤
1.  **更新 CSS**: 在 `updateStyles` 函数中添加针对 `.mode-settings` 等状态类的 CSS 规则，并修改 Modal 的基础样式。
2.  **更新 JS**: 修改 `configBtn.addEventListener` (打开设置)、`cancelBtn`/`saveBtn` (关闭设置) 以及 Edit/Delete 相关的事件处理函数，加入 `classList` 操作。

## 预期效果
打开设置时，界面内容平滑切换，背景保持单层毛玻璃效果，通透且清晰，不再有“双重叠加”导致的浑浊感。
