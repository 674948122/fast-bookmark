## 问题分析
用户反馈高亮色配置未生效。通过检查 [content.js](file:///Users/pandegong/my-project/fast-bookmark/content.js) 的代码，发现虽然 `--primary-color` 已正确配置为动态值，但 `--selected-bg` 和 `--hover-bg` 仍然是硬编码的颜色值。

具体问题代码（第 63-64 行）：
```javascript
        --hover-bg: ${isDark ? "#1f2937" : "#f3f4f6"};
        --selected-bg: ${isDark ? "#312e81" : "#e0e7ff"};
```
这里的 `#312e81` 和 `#e0e7ff` 是基于默认靛蓝色 (`#3730a3`) 的衍生色，它们并没有随着用户自定义的高亮色（`--primary-color`）而变化。因此，即使用户修改了高亮色，选中项的背景色（`fast-bookmark-selected` 类使用 `--selected-bg`）和悬停背景色仍然保持不变，导致用户感觉配置未生效。

## 实施方案

### 1. 动态生成衍生色
为了让选中背景色和悬停背景色跟随主色变化，我们需要使用 `color-mix` CSS 函数或者在 JS 中计算衍生色。考虑到兼容性和简便性，我们可以使用 CSS 的 `color-mix()` 函数（现代浏览器支持良好）或者简单的透明度叠加。

**方案 A：使用 CSS `color-mix`（推荐）**
```css
--selected-bg: color-mix(in srgb, var(--primary-color), transparent 85%);
--hover-bg: color-mix(in srgb, var(--primary-color), transparent 92%);
```
或者，如果不想依赖 `color-mix`，可以使用 RGB 变量拆分法，或者直接在 JS 中使用十六进制转 RGB 并设置透明度。

鉴于当前代码是在 JS 模板字符串中生成的 CSS，我们可以直接在 JS 中处理颜色。由于 `settings.highlightColorLight/Dark` 是十六进制颜色，我们可以编写一个简单的辅助函数将其转换为带有透明度的 RGB/RGBA 值，或者利用 `color-mix`。

考虑到 Chrome 扩展的环境，现代 CSS 特性 `color-mix` 是可用的。

### 2. 修改 `updateStyles` 函数
- 将 `--selected-bg` 修改为基于 `--primary-color` 的半透明版本。
  - 例如：`color-mix(in srgb, var(--primary-color), transparent 80%)`（即 20% 不透明度）
- 将 `--hover-bg` 修改为更浅的半透明版本。
  - 例如：`color-mix(in srgb, var(--primary-color), transparent 90%)`（即 10% 不透明度）
- 另外，需要注意在暗黑模式下，背景色可能需要不同的混合策略（例如混合黑色或白色，或者调整透明度）。
- 为了确保效果，我们可以直接使用 `color-mix` 混合背景色和主色。
  - `--selected-bg`: `color-mix(in srgb, var(--primary-color) 20%, var(--bg-color))`
  - `--hover-bg`: `color-mix(in srgb, var(--primary-color) 10%, var(--bg-color))`
  - 这样可以确保颜色在任何背景下都能正确显示。

  这部分逻辑可能也需要根据用户配置进行调整，或者确认是否应该保持独立。考虑到这是“搜索中”的状态色，可能应该保持原样，或者也允许配置。用户的问题主要在于“配置的高亮色未生效”，通常指的是默认状态下的选中/高亮。

```javascript
        --hover-bg: color-mix(in srgb, var(--primary-color) ${isDark ? '15%' : '10%'}, var(--bg-color));
        --selected-bg: color-mix(in srgb, var(--primary-color) ${isDark ? '25%' : '15%'}, var(--bg-color));
```
