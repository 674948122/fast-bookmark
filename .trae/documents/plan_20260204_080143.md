## 问题分析
用户反馈 `--accent-color` 没有同步更新。经过检查 [content.js](file:///Users/pandegong/my-project/fast-bookmark/content.js)，发现存在以下两个问题：

1. **`.fast-bookmark-searching` 状态下的硬编码覆盖**（第 73-75 行）：
   ```javascript
      :host(.fast-bookmark-searching) {
        --accent-color: ${isDark ? "#f59e0b" : "#6d28d9"};
      }
   ```
   这段代码在搜索状态下强制将 `--accent-color` 设置为固定的黄色或紫色，覆盖了用户配置的 `--primary-color`。这意味着当用户进行搜索时，高亮色（如匹配文字的高亮、路径徽章等）会恢复为默认颜色，而不是用户配置的颜色。

2. **`.fast-bookmark-folder-icon` 的硬编码颜色**（第 346 行）：
   ```javascript
      .fast-bookmark-folder-icon {
        color: #f59e0b; /* Slightly warmer amber */
        display: flex;
      }
   ```
   文件夹图标的颜色被硬编码为琥珀色，没有使用 `var(--accent-color)` 或 `var(--primary-color)`，因此也不会跟随用户配置变化。

3. **`.fast-bookmark-highlight` 背景色硬编码**（第 414 行）：
   ```javascript
        background: ${isDark ? "rgba(245, 158, 11, 0.15)" : "rgba(245, 158, 11, 0.12)"};
   ```
   高亮文字的背景色是基于固定颜色的透明度，应该修改为基于 `var(--accent-color)` 的动态混合。

## 实施方案

### 1. 移除硬编码覆盖
- 删除 `:host(.fast-bookmark-searching)` 规则块，确保 `--accent-color` 始终继承自 `--primary-color`（即用户配置的颜色）。
- 修改 `updateStyles` 中的定义，确保 `--accent-color` 始终等于 `--primary-color`。

### 2. 更新文件夹图标颜色
- 将 `.fast-bookmark-folder-icon` 的 `color` 属性修改为 `var(--accent-color)`，使其跟随主题色。

### 3. 更新高亮文字背景色
- 将 `.fast-bookmark-highlight` 的 `background` 属性修改为使用 `color-mix`，基于 `var(--accent-color)` 生成背景色。
  ```css
  background: color-mix(in srgb, var(--accent-color) ${isDark ? "15%" : "12%"}, transparent);
  ```

### 4. 验证其他使用点
- 检查 `.fast-bookmark-result-path`（第 378 行），它使用了 `var(--accent-color)`，这是正确的。但其背景色（第 379 行）也是硬编码的灰度背景，可以考虑调整为淡色主题色背景，或者保持现状（灰度背景配主题色边框也不错，先保持现状或微调）。
- 检查其他 `var(--accent-color)` 的使用，确保逻辑一致。
